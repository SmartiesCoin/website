Tokenomics Explained

How SMT emissions and masternode rewards work on mainnet.

Core Parameters

  • Max supply: 100,000,000 SMT (theoretical; ~90M without governance proposals)
  • Base block subsidy: 50 SMT
  • Halving interval: 1,000,000 blocks (since v0.1.4, block 40,000+)
  • Target block time: 60 seconds
  • Approx. halving cadence: ~694 days

Emission decreases through halvings. Since v0.1.4 (block 40,000), the reward split is fixed at 45% miner / 45% masternode / 10% treasury.

Reward Formula (since v0.1.4)

Every block (from block 40,000):

Treasury reserve = block subsidy x 10%

Distributable = block subsidy - treasury reserve

MN reward = distributable x 50% (fixed)

Miner reward = distributable x 50% (fixed)

The 10% treasury reserve is deducted from every block. It is only paid out in superblocks (every 21,600 blocks) if governance proposals are approved. If no proposals are approved, those coins are not created, reducing the effective supply. The 45/45/10 split is fixed and no longer changes with block height.

Treasury & Governance (since v0.1.3)

10% of the block subsidy is reserved from every block. These reserved coins are only paid out in superblocks (every 21,600 blocks / ~15 days) when governance proposals are approved by masternode vote.

  • Treasury reserve: 10% of every block subsidy
  • Superblock cycle: 21,600 blocks (~15 days)
  • Proposal cost: 5 SMT (anti-spam)
  • No approved proposals? Reserved coins are not created, reducing effective emission

Anyone can submit a funding proposal. Masternode operators vote to approve or reject it. Approved proposals are paid automatically by the network in the next superblock. If no proposals are approved, the treasury portion is never minted, resulting in a lower actual supply than the theoretical max.

Why Rewards Change Over Time

SMT uses two timing layers (since v0.1.4):

  • Halving clock: reduces subsidy every 1,000,000 blocks (~694 days).
  • Treasury reserve: 10% reserved every block; paid only if governance proposals pass.

The 45/45/10 split is fixed. Only halvings change the absolute payout amounts. Without approved proposals, the treasury portion is never created.

Concrete Examples (v0.1.4)

  • Normal block: 50 base - 5 treasury = 45 → 22.5 MN / 22.5 miner
  • Superblock w/ proposal: 5 SMT paid to proposal + 22.5 MN / 22.5 miner
  • Superblock w/o proposal: 5 SMT not created → 22.5 MN / 22.5 miner
  • After 1st halving: 25 base - 2.5 treasury = 22.5 → 11.25 MN / 11.25 miner

These are base examples without fee variation. The 10% treasury reserve applies to every block. Actual max supply depends on governance activity (~90M without proposals, up to 100M with full governance).
